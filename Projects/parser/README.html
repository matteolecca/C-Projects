PARSER

A parser is a component of a compiler or interpreter, which parses the source code of a computer programming language to create some form of internal representation; the parser is a key step in the compiler frontend. Programming languages tend to be specified in terms of a deterministic context-free grammar because fast and efficient parsers can be written for them. Run the program

To run the program:

  • Download the entire parsing folder.
  • From the terminal navigate to the folder location.
  • Run make -f nal.mk to generate the executables.
  • To test the interpreter run make -f nal.mk testinterp
  • To test the parser run make -f nal.mk testparse
  • Multiple text/nal files can be found into the folder.
  • To run a single test:
    • Run make -f nal.mk to genertate the executbles
    • to test the parser run e.g. ./parse test1.nal
    • to test the interpreter run e.g. ./interp test1.nal
